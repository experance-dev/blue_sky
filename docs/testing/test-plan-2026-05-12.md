This is the executable plan. Each section enumerates the tests we'll add, the tests we'll fix, and the projected per-class coverage after execution.

Legend:

## Order of operations (blocking dependencies)

```
Step 1  ── Fix permset metadata (Cluster A)
Step 2  ── Fix Jest modal-mock infra (Clusters F, G, H)
Step 3  ── Build EngagementTestFixtures.cls
Step 4  ── Refactor existing tests to use EngagementTestFixtures (in 3-5 file batches)
Step 5  ── Add new tests by class, in this priority order:
            5a. EngagementSignalDecayBatch  (worst coverage — 20%)
            5b. EngagementAdminController  (60% — admin tools)
            5c. LeadEngagementReparentHandler  (65%)
            5d. ContactEngagementErasureHandler + LeadEngagementErasureHandler  (67-70%)
            5e. EngagementController  (73%)
            5f. EngagementInboundRest  (78%)
            5g. EngagementErasureService  (79%)
            5h. EngagementSignalRouter  (84%)
            5i. EngagementServiceImpl  (currently 0% due to setup-blocked; flips to ≥90% on permset fix + a few new bulks)
Step 6  ── Add bulk/governor-limit stress tests across the board
Step 7  ── Add USER_MODE permission-denial tests
Step 8  ── Verify; capture coverage; append to DEMO.md
```

**Step 1 unblocks ~30 failing tests with zero new test code.** Steps 2–4 are infrastructure. Steps 5+ add the coverage to hit ≥90% per tier.

**Total estimated implementation time: 6–8 hours.**
