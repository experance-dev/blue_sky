**Purpose:** documents the pre-existing failures CI accepts in `dwood_z` when running `RunLocalTests`. CI gate logic:

| Outcome                   | Gate                                                            |
| ------------------------- | --------------------------------------------------------------- |
| Fail not in canon         | 🟥 BLOCK — new regression                                       |
| Fail in canon             | 🟦 passes — acknowledged drift                                  |
| Canonized test now PASSES | 🟧 flag — canon needs pruning; PR description must note removal |
| All pass + canon empty    | 🟢 release-ready (the goal)                                     |

## §D. Flake watch

`UNABLE_TO_LOCK_ROW` failures originally totaled 37/65 entries and were stochastic — across six recent runs of the same branch tip the lock-row failure count varied from ~10 to 37. The 2026-05-16 BSKY-201 refactor closed 32 of those 37 by making Portal\_\* test data unique per test method (Username/Account-Name/Contact-Email all tagged with a `Crypto.getRandomLong()`-derived suffix), eliminating the org-wide-uniqueness row-lock contention without sacrificing parallel test throughput. **Residual 5 rows** in `CaseTriggerHandlerTest` + `ContactServiceTest` remain canon-acknowledged; they're owned by the Zelis IE team and apply the same pattern.

## §E. Maintenance protocol

Per [§13.3 — Adding to the canon requires Atlas sign-off](../standards/sf-best-practices.md#133--adding-to-the-canon-requires-atlas-sign-off).

1. **Adding a row:** Pippa or Verity files a BSKY ticket → categorizes per §A → drafts the row → Atlas signs off → row lands.
2. **Removing a row:** when a previously-failing test now passes, CI flags 🟧. The PR removing the row must reference the fix commit and pass the same-named test in its own pre-flight run.
3. **Re-baselining:** if a sandbox refresh or major Zelis-legacy fix lands, regenerate from a fresh `RunLocalTests` run and diff against this file. Net additions or net subtractions both require a §F change-log entry.
